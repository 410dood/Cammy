## Cammy 2026 Final Competitor-Refresh Report (combined)

### Executive summary

Cammy is at or beyond feature parity with the self-hosted NVR field on classic detection, recording, analytics, and security (#1-#70 commercial + R01-R24 residential: object tracking, line-crossing, occupancy, heatmaps, cross-camera Re-ID, TOTP 2FA, per-camera RBAC, forward-auth SSO, offsite S3 backup, native Web Push, and an assistive-safety suite). The earlier note that "no competitor candidates surfaced" was an artifact of a failed first research run â€” this refresh recovers the competitor half and changes the picture: the 2025-2026 field has moved to **AI-native, local-LLM experiences** (Frigate 0.17 in-app trainable MobileNetV2 classifiers; Agent DVR "Ask AI" VLM alert-gating + feedback learning; Google "Ask Home", Reolink Smart Summary, Wyze Descriptive Alerts; Bosch IVA Pro Context, Coram Discover), **cross-camera identity/journey fusion** (Apple iOS 27 Home, Synology DVA, Verkada Unified Timeline), **ecosystem bridging** (Scrypted HomeKit/Matter; native Home Assistant config-flow; Matter 1.5 cameras Nov 2025), and **edge NPU acceleration** (Frigate OpenVINO/RKNPU; YOLO-World/YOLOE open-vocab detection). Cammy already ships the local primitives for most of these (CLIP crop embeddings, the #38 GenAI captioner, the SORT-lite tracker, go2rtc/ONNX, MQTT/HA discovery) but **composes none of them into the AI-native experiences competitors now charge subscriptions for** â€” which is precisely the no-subscription local-first wedge. The headline competitor-driven opportunities are: a VLM alert-verification gate, a per-camera false-alert learning loop, caption-aware notifications, upload-a-photo + CLIP attribute search, cross-camera journey fusion, a conversational "ask your cameras" layer, and an OpenVINO execution provider. The headline internal-hardening opportunities are unchanged and still essential: the **silent-failure / silent-miss cluster** (un-downloaded models, audio dead-windows, dead GenAI/push links, uncovered safety cameras all degrade with zero user signal â€” undercutting the liability-framed safety thesis), the **single-detection-thread scale ceiling** vs Blue Iris's 64-camera claim, **never-live-benchmarked cross-OS GPU** (CoreML/CUDA), and a **missing production deployment/Docker path** that is the single biggest adoption blocker vs the field.

### Competitor-driven feature gaps

#### Theme 1 â€” AI-native: local LLM/VLM verification, descriptive alerts, conversational Q&A, feedback learning

The defining 2025-2026 move and Cammy's biggest strategic exposure. Every competitor experience here is subscription-gated (Ring $30/mo, Wyze $20/mo, Google Home Premium) â€” the exact wedge for a free local NVR. Cammy ships the #38 VLM client and CLIP embeddings but uses them passively.

- **VLM alert-verification gate** (M). Agent DVR "Ask AI" (v6.5.3.0+, Ollama/vLLM/LM Studio yes/no filtering); Bosch IVA Pro Context (edge VLM verifier, 2026). Today the alarm fires first (pipeline.rs:868) and the caption job dispatches after, off-thread â€” so the VLM can never influence whether the rule fires. Add `vlm_prompt`/`vlm_expect` to AlarmRule (ride schedule_json, no migration) and a `vlm_ok()` AND-ed at the alarm site exactly like `confirm_ok`. **Caveats: fail-OPEN on any error/timeout (mirror confirm_ok's unwrap_or(true)); never gate life-safety on it; the multi-second call cannot run inline â€” defer the gated fire to a bounded worker so detection never stalls.**
- **Per-camera false-alert learning loop** (M). Agent DVR "Learning From Feedback" (2025-26). A thumbs-down on an event copies its `crop_embedding` into a new per-camera `rejected_crops` store; at emit time, cosine-compare a fresh detection's crop against that camera's rejected set and suppress when within threshold of â‰¥N examples. **Caveats: fail-OPEN (never suppress on error, per the confirm_ok convention); cap the set, keep threshold conservative/tunable, add a "Suppressed" view + per-camera Clear-Learning, log suppressions for audit.** Zero new model â€” strong local-first differentiator and the cleanest attack on alert fatigue.
- **Caption-aware push notifications** (M). Wyze Descriptive Alerts; Google AI Notifications; Arlo Event Captions (all put the NL description IN the push). The #38 caption reaches only the post-hoc anomaly notification today; the live ntfy/webhook/MQTT/email/WebPush paths carry label/face/plate/gesture/transcript but no caption. Add `caption` to AlarmEvent + a `{{caption}}` placeholder (same pattern #46 used for transcript). **Solve the 1-60s async timing without blocking detection:** have the genai worker re-fire/enrich the alert (or write a caption-bearing notifications row #68 drains, mirroring anomaly.rs), behind a per-rule "describe in notification" toggle with label fallback.
- **Conversational "ask your cameras"** (L). Google Ask Home / Gemini for Home (2026); Coram Discover; Spot AI; TP-Link Aireal (CES 2026); Reolink Smart Summary; Milestone Video Summarization (2026). A bounded read-only tool-calling loop exposing ONLY query tools (events filters, CLIP search, transcripts, captions, counts/occupancy/Re-ID) with strict iteration caps + a chat panel rendering tappable event citations. **Caveats: needs a separate tool-calling chat model (the #38 model is a vision captioner) â€” add `assistant_model`, degrade gracefully when absent; treat ALL tool outputs (attacker-influenceable transcripts/captions/notes) as untrusted data, expose NO mutating tools, live-validate prompt-injection resistance; off by default like every LLM feature.**
- **In-app user-trainable classifiers (MobileNetV2 transfer-learning)** (XL). Frigate 0.17 custom classification (state + object, MobileNetV2, CPU); 0.16 bird-species; 0.17 attribute (helmet/hi-vis/leash). The "third compute tier" between the motion gate and full YOLO/CLIP â€” and the substrate that collapses four competitor candidates (object classification, fixed-region STATE classifier as an MQTT/HA sensor, species/breed, per-object PPE/leash) onto one engine. Ship a bundled feature-extractor ONNX (downloaded-not-committed), an in-Rust linear/logistic head trainer (ndarray, no heavy dep), new DB tables for classes/samples/trained-head weights, a "Classification" crop-bucketing UI, and an inference hook through the existing event/alarm/MQTT-HA paths. **Scope honestly as XL; land the STATE-classifier vertical (garage/bin open-vs-closed â†’ HA binary_sensor) first; train strictly off the shared detection thread (spawn_blocking).** This is the PPE slice's only responsible path under the no-new-model bar.

### Industry trends 2026

- **User-trainable LOCAL models are the defining self-hosted-NVR move** (Frigate 0.17 in-app MobileNetV2 transfer-learning, no cloud). The field shifted from fixed COCO labels to "recognize MY scene"; Cammy's all-fixed-model stack is now the outlier.
- **A three-tier compute hierarchy is standard:** motion gate â†’ general GPU detector â†’ ultra-light MobileNetV2 classifiers for narrow recurring questions. Cammy has only two tiers.
- **Local VLM/LLM as a second-stage VERIFIER and CONVERSATIONAL layer** is the breakout, consumer (Wyze, Reolink, Google Ask Home, TP-Link Aireal) and enterprise (Bosch IVA Pro Context, Milestone, Genetec, Verkada, Coram, Spot AI) â€” several now run the VLM on-prem/at the edge, validating Cammy's local-first thesis. Cammy's #38 captions are passive (no gate, no dialogue).
- **Anti-fatigue is solved semantically:** descriptive notification TEXT, per-camera online learning from confirm/reject feedback, and severity-tier suppress/merge triage (Wyze NBD 1-5, Reolink L1-L4). Cammy's cooldown (#30) is the blunter, non-semantic version.
- **Cross-camera identity + journey FUSION is the new flagship analytic** (Apple iOS 27 Home, Synology DVA, Verkada/Genetec map trajectories). Cammy has the Re-ID + tracker primitives but never composes them.
- **Forensic review is shifting to NL + ATTRIBUTE search** (clothing/vehicle color+type, upload-a-photo, find-repeated-objects) with confidence sliders, license-free and local (Synology/QNAP/UniFi/Apple).
- **Smart-home interoperability is the dominant differentiator** (Scrypted is built around it); the expected surface moved from raw MQTT to a NATIVE Home Assistant integration, and WebRTC became the universal transport (HA bundles go2rtc; Matter 1.5 standardized cameras Nov 2025; Echo Show/Nest Hub mandate WebRTC). Cammy's go2rtc stack is well-positioned but exposes none of these.
- **NPU acceleration via ort EPs** (OpenVINO for Intel Core Ultra, RKNPU for RK3588) is the 2026 edge-AI story; enrichment models increasingly pinned to their OWN accelerator (exactly Cammy's documented CLIP-stalls-the-detector workaround). Open-vocab detection (YOLO-World/YOLOE) and the post-YOLOv8 frontier (YOLO11/RF-DETR) break the fixed-80-class ceiling.
- **Proactive on-site DETERRENCE** (Verkada escalating speaker talk-down) reframes two-way audio from manual convenience to automated alarm action.
- **Export redaction is table-stakes** (Milestone Anonymization, Genetec, Sighthound Redactor) â€” distinct from live masks and from tamper-evident export.
- **Reliability + container-first deployment are taken for granted** (Docker/Compose + NAS app catalogs primary; desktop installers secondary; SMART/UPS, Prometheus+Grafana). Cammy has no Dockerfile/compose â€” the single biggest adoption blocker.
- **Every conversational/descriptive/triage feature is subscription-gated across the consumer field** (Ring $30/mo, Wyze $20/mo, Google Home Premium) â€” the precise wedge for a no-subscription local NVR shipping the same AI experiences for free.
